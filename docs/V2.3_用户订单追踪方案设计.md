## 一、背景与目标

### 1.1 当前系统的缺失

v2.1 → v2.2 构建了一套完整的**自动化市场分析系统**：

```
Market Data → Indicator Engine → Signal Engine → DirectionTracker → Decision Trace
```

但整个系统完全**面向市场**，不感知用户的交易行为：

| 缺失项 | 影响 |
|--------|------|
| 系统不知道用户何时入场 | 无法评估用户的交易决策质量 |
| 系统不知道用户的持仓成本 | 无法在关键价位给出提示 |
| 用户只能看"市场信号"，没有"个人信号" | 信号缺乏针对性和行动指引 |

### 1.2 V2.3 要解决的问题

| 问题 | 现状 | V2.3 目标 |
|------|------|-----------|
| 用户入场价格记录 | 没有任何记录 | 用户在监控列表页一键记录开单（多/空方向可选） |
| 个人关键价位 | 只有订单簿的大单价位 | 用户的开单价格成为一个个人的支撑/压力位 |
| 价位回测决策 | 无 | 当价格回到入场价，结合系统信号给出建议 |
| 原始订单流数据 | 低频使用且有噪音 | 改造为订单管理界面 |

### 数据流

```
用户行为                                系统行为
─────────                            ─────────
用户开单 BTC LONG                    记录到 orders 表
@ $64,500                            存入 user_trades.db
      │
      ▼
价格波动...                           每 5s 轮询行情
      │
      ▼
价格回到 $64,500附近                  触发 price_level 检测
      │                              比较当前方向 vs 开单方向
      │                              方向一致 → "持有"
      │                              方向反转 → "考虑平仓"
      │
      ▼
用户接受到提示 → 决定平仓或持有
      │
用户平仓 → 从 orders 移至 closed_orders
```

### 4.2 价格触及检测器

`backend/app/services/price_level_tracker.py`

```python
class PriceLevelTracker:
    """
    价格触及检测 + 系统信号联动。
    
    当最新价接近某个开单价格时：
      1. 获取当前系统评分/方向/置信度
      2. 比较当前方向与用户开单方向
      3. 生成建议（持有/考虑平仓）
      4. 如果有重大变化，通过 Event Bus 触发告警
    """
    
    def check_levels(symbol, current_price, analysis_result) -> list:
        """
        返回所有触及的开单 + 系统建议。
        
        Returns:
          [
            {
              "order_id": 1,
              "symbol": "BTC-USDT",
              "entry_price": 64317.2,
              "direction": "LONG",
              "distance_pct": 0.12,      # 当前价偏离%
              "system_direction": "SHORT",
              "system_score": 78,
              "system_confidence": 82,
              "suggestion": "consider_close",  # hold / consider_close
              "reason": "系统当前方向 SHORT，与你的开单方向 LONG 相反"
            }
          ]
        """
```

### 4.3 触发条件

| 条件 | 行为 |
|------|------|
| 最新价在开单价 ±0.5% 范围内 | 触发检测，与系统信号联动 |
| 系统方向与开单方向**一致** | `suggestion: "hold"` |
| 系统方向与开单方向**相反** | `suggestion: "consider_close"` |
| 系统方向为 NEUTRAL 或置信度 < 50 | `suggestion: "observe"`（建议观察） |

## 五、数据库设计

### 5.1 user_trades.db（新增独立库）

与原 `signals.db` 分离，避免混合。

#### orders 表（当前持仓）

```sql
CREATE TABLE IF NOT EXISTS orders (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol TEXT NOT NULL,              -- BTC-USDT
    direction TEXT NOT NULL,           -- LONG / SHORT
    entry_price REAL NOT NULL,         -- 开仓价格
    quantity REAL DEFAULT 0,           -- 数量（用户填写）
    note TEXT DEFAULT '',              -- 备注
    status TEXT DEFAULT 'open',        -- open / closed
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX IF NOT EXISTS idx_orders_status ON orders(status);
CREATE INDEX IF NOT EXISTS idx_orders_symbol ON orders(symbol);
```

#### closed_orders 表（已平仓归档）

```sql
CREATE TABLE IF NOT EXISTS closed_orders (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    original_order_id INTEGER,          -- 原订单 ID（可选）
    symbol TEXT NOT NULL,
    direction TEXT NOT NULL,
    entry_price REAL NOT NULL,
    close_price REAL NOT NULL,          -- 平仓价格
    quantity REAL DEFAULT 0,
    pnl_pct REAL,                       -- 盈亏百分比
    pnl_abs REAL,                       -- 绝对盈亏
    result TEXT,                        -- win / loss / breakeven
    note TEXT DEFAULT '',
    opened_at TEXT NOT NULL,             -- 开仓时间
    closed_at TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX IF NOT EXISTS idx_closed_symbol ON closed_orders(symbol);
CREATE INDEX IF NOT EXISTS idx_closed_result ON closed_orders(result);
```

### 5.2 盈亏计算

```python
def calc_pnl(direction, entry_price, close_price, quantity):
    if direction == "LONG":
        pnl_pct = (close_price - entry_price) / entry_price * 100
    else:  # SHORT
        pnl_pct = (entry_price - close_price) / entry_price * 100
    
    pnl_abs = pnl_pct / 100 * entry_price * quantity
    
    if pnl_pct > 0.1:
        result = "win"
    elif pnl_pct < -0.1:
        result = "loss"
    else:
        result = "breakeven"
    
    return round(pnl_pct, 2), round(pnl_abs, 2), result
```

## 七、价格触及信号逻辑

### 7.1 核心规则

```
当 |current_price - entry_price| / entry_price < threshold_pct
    系统方向 == 开单方向 → suggestion: "hold"（持有）
    系统方向 != 开单方向 → suggestion: "consider_close"（考虑平仓）
    系统方向 == NEUTRAL 或 置信度 < 50 → suggestion: "observe"（继续观察）
```

### 7.2 阈值参数

| 参数 | 默认值 | 说明 |
|:----:|:-----:|------|
| threshold_pct | 0.5% | 触及检测范围（价格在开单价的 ±0.5% 内即触发） |
| hold_confidence_min | 60 | 建议"持有"所需的最低置信度 |
| cooldown_minutes | 30 | 同个订单的触及提示冷却时间（防重复提醒） |

### 7.3 触发场景示例

```
场景 1：BTC SHORT 开仓 $64,500，价格回到 $64,200
  触及范围：$64,177.5 ~ $64,822.5（±0.5%）
  当前系统评分：SHORT 78 分
  → 方向一致 → "持有" ✅

场景 2：ETH LONG 开仓 $3,500，价格回到 $3,512
  触及范围：$3,482.5 ~ $3,517.5
  当前系统评分：SHORT 65 分
  → 方向相反 → "考虑平仓" ⚠

场景 3：SOL LONG 开仓 $142，价格回到 $141.3
  触及范围：$141.29 ~ $142.71
  当前系统评分：NEUTRAL 38 分，置信度 42%
  → 置信度 < 50 → "继续观察" 🔍
```

## 附录 B：UI 修复与模块评估（V2.3 附带）

根据对现有前端界面的逐项审查，发现以下问题并确认修复方案。

### B.1 价格自适应显示

**现状：** `fmtNumber(value, digits=2)` 固定 2 位小数。低单价币种（如 <$1）无法显示有效精度。

**修复方案：** 根据价格量级自适应小数位数

| 价格区间 | 小数位数 | 示例 |
|:--------:|:-------:|------|
| ≥ 1000 | 2 位 | $64,317.20 |
| 1 ~ 999 | 4 位 | $512.5000 |
| 0.01 ~ 0.999 | 6 位 | $0.052300 |
| < 0.01 | 8 位 | $0.00123400 |

```javascript
export function fmtNumber(value, digits) {
  if (value === null || value === undefined || Number.isNaN(Number(value))) return "-";
  // 自适应小数位
  if (digits !== undefined) {
    // 显式指定时使用指定值
  } else {
    const abs = Math.abs(Number(value));
    if (abs >= 1000) digits = 2;
    else if (abs >= 1) digits = 4;
    else if (abs >= 0.01) digits = 6;
    else digits = 8;
  }
  return Number(value).toLocaleString(undefined, {
    minimumFractionDigits: digits,
    maximumFractionDigits: digits,
  });
}
```
