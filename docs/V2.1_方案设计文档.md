## 一、v2.0 核心问题

### 1.1 评分永远达不到 100 分

v2.0 的 [market_score.py](file:///d:/OKEworkplace/backend/app/model/market_score.py) 采用四维评分：

```
Trend(0~25) + Capital(0~25) + OrderFlow(0~25) + Risk(-25~0) = 总分
                                           ↑ Risk 永远是 0 或负数
```

理论满分 = 25 + 25 + 25 + 0 = **75**，实际行情中约 **40~50**。这是结构性缺陷，不是参数问题。

### 1.2 两套评分系统并存冲突

| 模块 | 评分方式 | 输出 |
|------|---------|------|
| `trading_rules.py` | 双向偏置分 (-∞ ~ +∞) | STRONG_BUY / BUY / SELL 等 |
| `market_score.py` | 0-100 四维评分 | 强势 / 震荡 / 弱势 |

两个系统各自独立、格式不同、权重不同，前端无法统一展示。

### 1.3 缺少关键指标

| 指标 | 状态 | 影响 |
|------|------|------|
| RSI(14) | 缺失 | 无法判断超买超卖 |
| ATR(14) | 缺失 | 风险评估无波动率基准 |
| EMA 金叉/死叉 | 未显式标记 | 趋势判断缺关键信号 |
| EMA(20/60) | 用了 SMA 替代 | 与主流分析工具不一致 |
| 做空评分 | 零散不可靠 | 程序实际只擅长做多分析 |

## 二、v2.1 核心设计：整合对称评分

### 2.1 设计理念

> **不追求 100 分开单。分数 = 信号质量，不是开单触发器。**

| 概念 | 说明 |
|------|------|
| 分数 | 当前方向上的信号质量（0-100）。60 分以上值得关注，80 分以上优质信号 |
| 方向 | LONG / SHORT / NEUTRAL，由趋势指标独立判定 |
| 对称性 | 做多和做空的评分逻辑完全镜像，同等对待 |

### 2.2 为什么是整合而非分离

```
分离方案（两套分数）：
  多头分: 78        空头分: 25        → 用户需要比较两个数字，容易困惑

整合方案（一个分数 + 方向）：
  score: 78, direction: LONG         → 一眼看清：值得关注，方向做多
  score: 78, direction: SHORT        → 一眼看清：值得关注，方向做空
```

成交量放大、OI 骤增、资金费率极端 — 这些是**方向中性的强度信号**，多空都在用。分离方案意味着每个维度都要写两遍，且中性指标无法合理分配到两边。

### 2.3 评分流程

```
输入数据（K线 / OI / 费率 / 成交量）
         │
         ▼
  第 1 步：方向判定
  ┌─────────────────────────────────┐
  │ EMA 排列 + RSI 区间 + MACD 方向  │
  │         + OI 方向验证            │
  │              ↓                   │
  │    LONG / SHORT / NEUTRAL       │
  └─────────────────────────────────┘
         │
         ▼
  第 2 步：质量评分（0-100）
  ┌─────────────────────────────────┐
  │  在判定方向上，各维度分别打分：    │
  │  趋势(30) + 动量(20) + 成交量(20) │
  │      + OI(20) + 资金费率(10)      │
  │              ↓                   │
  │         0 ~ 100                  │
  └─────────────────────────────────┘
         │
         ▼
  第 3 步：风险评级 + 解释生成
  ┌─────────────────────────────────┐
  │  ATR 波动率 + 支撑阻力距离        │
  │         + 资金费率极端             │
  │              ↓                   │
  │   LOW / MEDIUM / HIGH           │
  └─────────────────────────────────┘
         │
         ▼
  统一输出 JSON
```

## 三、评分公式详述

### 3.1 总分构成

```
总分 = 趋势(30) + 动量(20) + 成交量(20) + OI(20) + 资金费率(10) = 0~100
```

### 3.2 趋势评分（满分 30）

**判向优先**：先用趋势指标确定 LONG/SHORT，再在主方向上打分。

#### 3.2.1 方向判定（独立于评分）

```
LONG 条件（需同时满足以下中 3/4）：
  ☑ EMA20 > EMA60（金叉状态）
  ☑ RSI 在 45~70 区间（非超卖）
  ☑ MACD DIF > DEA（多头排列）
  ☑ OI 增加或持平（资金未撤离）

SHORT 条件（镜像）：
  ☑ EMA20 < EMA60（死叉状态）
  ☑ RSI 在 30~55 区间（非超买）
  ☑ MACD DIF < DEA（空头排列）
  ☑ OI 增加或持平（空头建仓）

否则 → NEUTRAL
```

#### 3.2.2 趋势质量评分（0-30）

在执行方向上打分（以 LONG 为例，SHORT 镜像）：

| 子项 | 条件 | LONG 得分 | SHORT 镜像 |
|------|------|:--:|:--:|
| EMA 排列强度 | EMA20 远超 EMA60（差距 > 2%） | 12 | EMA20 远低于 EMA60 |
| | EMA20 > EMA60（差距 0.5~2%） | 8 | 镜像 |
| | EMA20 ≈ EMA60（差距 < 0.5%） | 4 | 镜像 |
| RSI 区间 | RSI 50~62（健康多头区） | 8 | RSI 38~50 |
| | RSI 62~68（偏强但有空间） | 5 | RSI 32~38 |
| | RSI > 72 或 < 30（极端，弱化） | 2 | 镜像 |
| MACD 状态 | DIF > DEA 且柱线 > 0 且柱线扩大 | 6 | 镜像反向 |
| | DIF > DEA 但柱线 > 0 且柱线缩小 | 4 | 镜像反向 |
| OI 验证 | OI 增长且与价格同向 | 4 | OI 增长 + 价格下跌 |
| | OI 持平 | 2 | 镜像 |

> LONG 满分 30 示例：EMA 多排明显 + RSI 58 + MACD 柱扩大 + OI 配合增仓

### 3.3 动量评分（满分 20）

| 子项 | 条件 | 得分 |
|------|------|:--:|
| K 线动量 | 最近 4 根 K 线中 ≥3 根顺势 | 8 |
| | 最近 4 根中 2 根顺势 | 4 |
| 价格位置 | 价格在 EMA20 的顺势侧 | 6 |
| 金叉/死叉 | 最近 3 根内出现 EMA 金叉/死叉 | 6 |
| | 最近 6 根内出现 EMA 金叉/死叉 | 3 |

> "顺势"：LONG 时指收阳线，SHORT 时指收阴线

### 3.4 成交量评分（满分 20）

此为方向中性维度，高分 = 市场活跃度高。

| 子项 | 条件 | 得分 |
|------|------|:--:|
| 量能强度 | 当前量 > 20 均量 × 2.0 | 10 |
| | 当前量 > 20 均量 × 1.5 | 7 |
| | 当前量 > 20 均量 × 1.2 | 4 |
| | 缩量（< 0.5 倍） | 0 |
| 量价配合 | 价顺势 + 量放大（健康） | 6 |
| | 价顺势 + 量萎缩（动能不足） | 2 |
| | 价逆势 + 量放大（背离警告） | 0 |
| 突破确认 | 突破 EMA20 且放量 > 1.3x | 4 |

### 3.5 OI 评分（满分 20）

| 子项 | 条件 | 得分 |
|------|------|:--:|
| OI 变化 | OI 变幅 > 5%（剧烈变化） | 8 |
| | OI 变幅 2~5% | 6 |
| | OI 变幅 0~2% | 3 |
| | OI 下降（资金流出） | 0 |
| 价-OI 配合 | 价顺势 + OI 增（资金推动） | 8 |
| | 价顺势 + OI 减（空头回补/多头止盈） | 4 |
| | 价逆势 + OI 增（对手盘建仓） | 0 |
| OI 趋势 | OI 连续 2 根同向 | 4 |

### 3.6 资金费率评分（满分 10）

资金费率是**反向指标**：极端值意味着拥挤交易。

| 子项 | 条件 | 得分 | 含义 |
|------|------|:--:|------|
| 做多时 | 费率 < -0.03%（空头拥挤 → 可能轧空） | 10 | 对手盘拥挤 = 有利 |
| | 费率 -0.01% ~ 0.02%（中性偏多） | 7 | 健康 |
| | 费率 0.02% ~ 0.05%（偏拥挤但可控） | 4 | 需警惕 |
| | 费率 > 0.05%（多头极度拥挤） | 1 | 高风险 |
| 做空时（镜像） | 费率 > 0.03%（多头拥挤 → 可能回调） | 10 | 对手盘拥挤 = 有利 |
| | 费率 -0.02% ~ 0.01%（中性偏空） | 7 | 健康 |
| | 费率 -0.05% ~ -0.02%（偏拥挤但可控） | 4 | 需警惕 |
| | 费率 < -0.05%（空头极度拥挤） | 1 | 高风险 |

### 3.7 评分等级

| 分数 | 等级 | 含义 |
|:----:|------|------|
| 85-100 | 🔥 强烈关注 | 多维度共振，高概率行情 |
| 70-84 | 📈 重点关注 | 结构良好，有交易价值 |
| 55-69 | 👀 观察 | 部分条件满足，值得跟踪 |
| 40-54 | ⚖ 中性 | 无明显方向性机会 |
| <40 | ❌ 放弃 | 信号混乱或市场沉寂 |

## 四、风险评估模块（独立于评分）

风险不参与评分加减，独立输出，供交易者决策参考。

```
风险等级 = f(ATR, 资金费率极端度, 支撑阻力距离, 方向一致性)
```

### 4.1 风险因子

| 因子 | LOW 条件 | HIGH 条件 |
|------|---------|----------|
| ATR/价格比 | < 1.5% | > 4%（剧烈波动） |
| 资金费率 | -0.02% ~ 0.03%（健康） | > 0.05% 或 < -0.05%（极端拥挤） |
| 距阻力位（做多时） | > 3% | < 1%（逼近阻力） |
| 距支撑位（做空时） | > 3% | < 1%（逼近支撑） |
| 指标一致性 | 3/4 指标同向 | 1/4 或 0/4 同向（混乱） |

### 4.2 风险输出示例

```json
{
  "risk": "MEDIUM",
  "risk_factors": [
    "ATR 偏高 (3.8%)",
    "距离阻力位较近 (1.2%)"
  ]
}
```

## 八、指标计算实现规范

### 8.1 RSI(14)

```
RSI = 100 - (100 / (1 + RS))
RS  = 最近 14 根 K 线的平均涨幅 / 平均跌幅

涨幅 = max(close - prev_close, 0)
跌幅 = max(prev_close - close, 0)
```

判断阈值：
- RSI > 70：超买
- RSI < 30：超卖
- RSI 50~65：多头健康区
- RSI 35~50：空头健康区

### 8.2 ATR(14)

```
TR = max(high-low, |high-prev_close|, |low-prev_close|)
ATR = EMA(TR, 14)
```

用途：
- 波动率评估（ATR / 价格）
- 风险等级判断
- 不做为止损建议（不由程序给具体价位）

### 8.3 EMA 金叉/死叉检测

      - cross_type: "golden" / "dead" / None

```

### 8.4 MACD 金叉/死叉检测

```
MACD(12, 26, 9)
DIF = EMA12 - EMA26
DEA = EMA(DIF, 9)
Histogram = DIF - DEA

金叉：DIF 上穿 DEA（当前 DIF>DEA 且前一根 DIF≤DEA）
死叉：DIF 下穿 DEA（当前 DIF<DEA 且前一根 DIF≥DEA）
```

## 十二、v2.2 预案：信号方向抖动控制

> **问题**：v2.1 每 5 秒轮询一次评分。在震荡行情中，评分可能在 LONG/SHORT/NEUTRAL 之间快速切换，例如：
> ```
> 12:00:05 → LONG  72
> 12:00:10 → SHORT 61     ← 刚才还是做多！
> 12:00:15 → LONG  68     ← 又变回做多了！
> 12:00:20 → NEUTRAL 43
> ```
> 这种"信号抽搐"导致用户恐慌、频繁交易、拿不住单。需要一套机制让方向判断"粘滞"且"稳重"。

### 12.1 设计目标

| 目标 | 说明 |
|------|------|
| 方向稳定 | 已判定的方向不会被噪音轻易翻转 |
| 及时响应 | 真正的趋势反转仍然能在合理时间内被识别 |
| 不压制质量 | 方向不变 ≠ 评分不更新。分数仍实时刷新 |
| 静默期 | 方向改变后有一段冷却期，期间不推送新方向提醒 |

### 12.2 三层过滤机制

```
┌──────────────────────────────────────────────┐
│            原始评分 (每 5s)                   │
│    score: 72, direction: LONG                 │
└──────────────────┬───────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────┐
│  第 1 层：趋势缓冲区 (Trend Signal Buffer)    │
│  ┌─────────────────────────────────────┐     │
│  │ 滚动记录最近12次方向判定（1分钟窗口）│     │
│  │ 输出：多数表决方向                   │     │
│  │ [L,L,L,L,L,    S,S,    L,N,   L,L] │     │
│  │  LONG=8, SHORT=2, NEUTRAL=2        │     │
│  │ → 表决方向: LONG                   │     │
│  └─────────────────────────────────────┘     │
└──────────────────┬───────────────────────────┘
                   │ 表决方向 ≠ 上次发布方向？
                   ▼
┌──────────────────────────────────────────────┐
│  第 2 层：迟滞带 (Hysteresis Band)            │
│                                              │
│  当前方向: LONG                               │
│  ┌──────────────────────────────────────┐    │
│  │  放弃区间          NEUTRAL 缓冲       │    │
│  │  score < 35    35 ───────── 55      │    │
│  │  可翻转为SHORT   可转为NEUTRAL       │    │
│  │                                      │    │
│  │  LONG 方向持续要求: score ≥ 55       │    │
│  │  SHORT 方向持续要求: score ≤ 35      │    │
│  │  NEUTRAL 区间: 35~55 (无方向)        │    │
│  └──────────────────────────────────────┘    │
│                                              │
│  LONG → NEUTRAL: 需表决方向=NEUTRAL 连续 3 次 │
│  LONG → SHORT : 需表决方向=SHORT  连续 4 次   │
│  NEUTRAL → LONG: 需表决方向=LONG   连续 3 次  │
│  NEUTRAL → SHORT: 需表决方向=SHORT  连续 3 次 │
└──────────────────┬───────────────────────────┘
                   │ 方向已改变？
                   ▼
┌──────────────────────────────────────────────┐
│  第 3 层：方向变更冷却 (Direction Cooldown)   │
│                                              │
│  方向变更后，N 分钟内不推送任何新方向提醒      │
│  默认冷却时间: 5 分钟                         │
│  可配置项: COOLDOWN_MINUTES                   │
│                                              │
│  冷却期间:                                    │
│    - 评分照常更新（用户仍能看到分数变化）      │
│    - 方向变更在冷却结束后才推送               │
│    - 如果冷却期间方向又变回来，不推送         │
└──────────────────────────────────────────────┘
```

### 12.3 关键参数设计

| 参数 | 默认值 | 含义 |
|------|:------:|------|
| `BUFFER_SIZE` | 12 | 趋势缓冲区窗口（12×5s = 60s） |
| `MAJORITY_RATIO` | 0.5 | 多数表决阈值（>50% 即有效） |
| `HYSTERESIS_LOW` | 35 | 迟滞带下限（低于此才考虑 SHORT） |
| `HYSTERESIS_HIGH` | 55 | 迟滞带上限（高于此才考虑 LONG） |
| `NEUTRAL_CONFIRM` | 3 | NEUTRAL 确认次数（连续 3 次 = 15s） |
| `DIRECTION_CONFIRM` | 3 | 方向确认次数（连续 3 次 = 15s） |
| `FLIP_CONFIRM` | 4 | 翻转确认次数（LONG→SHORT 需连续 4 次 = 20s） |
| `COOLDOWN_MINUTES` | 5 | 方向变更后冷却时间（分钟） |

### 12.4 效果模拟

**震荡行情（原 v2.1 会抽搐）**：

```
原始方向序列（每 5s）：
L, L, S, L, S, N, L, S, L, N, L, L

第 1 层 缓冲区表决：
  窗口1 (1-12): L×6, S×3, N×3 → 多数=L (6/12≥50%) → 表决: LONG
  窗口2 (2-13): L×5, S×3, N×4 → 多数=N (4/12<50%, L=5/12) → 表决: LONG (仍多数)

第 2 层 迟滞带：
  当前方向=LONG, 表决=LONG (连续N次) → 保持不变

最终发布方向: LONG（稳定）
```

**真实趋势反转**：

```
原始方向序列：
L, L, L, L,  L, L, S, S,  S, S, S, S
本周窗口: L×6, S×6 → 平票 → 表决: NEUTRAL
下个窗口: L×4, S×8 → 多数=S → 表决: SHORT

第 2 层 迟滞带：
  当前方向=LONG, 表决=SHORT, 连续次数=1, 需≥4 → 不触发
  ......
  连续 4 次表决=SHORT（共耗时 4×5s = 20s）→ 触发翻转

最终: LONG → SHORT（20 秒内确认真实趋势反转）
```

**冷却期效果**：

```
12:00:00  LONG  →  方向不变，无需冷却
12:05:30  SHORT →  方向改变！推送提醒，启动 5 分钟冷却
12:05:35  LONG  →  冷却中，不推送
12:06:00  SHORT →  冷却中，评分照常更新但不推送
12:07:00  LONG  →  冷却中
12:10:30  冷却结束
12:10:35  方向=LONG 已稳定 → 推送提醒，启动新冷却
```

### 12.5 数据结构设计

```python
@dataclass
class DirectionTracker:
    """方向跟踪器（每个币种一个实例）"""
    symbol: str
    
    # 趋势缓冲区
    buffer: deque  # 最近 N 个方向判定，maxlen=BUFFER_SIZE
    
    # 当前发布状态
    published_direction: str = "NEUTRAL"  # 已推送的方向
    published_score: int = 0              # 已推送的评分
    
    # 确认计数器
    direction_streak: int = 0             # 当前方向的连续次数
    streak_target: str = "NEUTRAL"        # 正在累计的方向
    
    # 冷却状态
    cooldown_until: float = 0.0           # 冷却结束时间戳
    pending_direction: str = None         # 冷却期间待推送的方向
    
    def feed(self, raw_direction: str, raw_score: int) -> dict:
        """
        输入原始判定，返回 {"should_notify": bool, "direction": str, "score": int}
        """
        # 第 1 层：写入缓冲区
        self.buffer.append(raw_direction)
        
        # 第 1 层：多数表决
        voted = self._majority_vote()
        
        # 第 2 层：迟滞带
        stable = self._hysteresis_check(voted)
        
        # 第 3 层：冷却
        if self._in_cooldown():
            if stable and stable != self.published_direction:
                self.pending_direction = stable
            return {"should_notify": False, "direction": self.published_direction, "score": raw_score}
        
        # 冷却外，方向稳定 → 推送
        if stable and stable != self.published_direction:
            self.published_direction = stable
            self._start_cooldown()
            return {"should_notify": True, "direction": stable, "score": raw_score}
        
        return {"should_notify": False, "direction": self.published_direction, "score": raw_score}
```
